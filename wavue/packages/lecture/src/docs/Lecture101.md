# Vue.js 안내

## 1. Vue를 학습하기 위해서는 Node.js가 필요합니다.
---

### - Node.js는 C#의 닷넷프레임워크나 Java의 JDK처럼, 개발을 위한 실행 환경이라고 생각하면 됩니다.
### - Vue.js 프로젝트를 생성하고 실행하기 위해서는 Node.js 설치가 반드시 필요합니다.
### - Node.js는 자바스크립트 언어를 사용합니다.
### - Node.js 환경에서는 자바스크립트가 브라우저 밖에서도 실행되며, 파일 시스템이나 네트워크 같은 PC 자원에도 접근할 수 있습니다.

## 2. Vue.js 란?
---
### - **Vue.js**는 브라우저에서 동작하는 프론트엔드 프레임워크입니다.  
### - Vue.js 프로젝트를 개발하고 실행하기 위해서는 Node.js가 필요합니다.  
### - Vue.js는 자바스크립트와 타입스크립트(Vue 3)를 기반으로 개발되었습니다.
### - Vue는 **HTML + CSS + JavaScript**를 효율적으로 사용하도록 설계되었습니다.  
### - 화면 전체를 새로고침하지 않고 필요한 부분만 빠르게 갱신할 수 있습니다.

## 1.개발환경설정
---
### - Node.js 설치 : Vue 개발을 위해 필수입니다. (npm도 함께 설치됨)
### - npm : Node.js에 기본 포함된 패키지 관리 도구입니다. (http://npmjs.com)
### - pnpm : npm과 동일한 역할을 하지만, 더 빠르고 효율적으로 동작하는 패키지 관리 도구입니다.
### - 모노레포(Monorepo) : 하나의 큰 프로젝트 안에서 여러 개의 하위 프로젝트(패키지)를 함께 관리하는 구조입니다.
### - Vue 프로젝트 설정  :Vue 프로젝트의 모든 설정은 `package.json`에서 관리합니다.
### - Vite 설정 : 빌드 도구로 사용됩니다. 설정 파일은 `vite.config.js` 또는 `vite.config.ts` 입니다.
### - 타입스크립트 설정 : 타입스크립트를 사용할 경우, `tsconfig.json` 파일을 통해 컴파일 옵션을 설정해야 합니다.

## 3. Vue UI 프레임워크
---

### “Vue UI 프레임워크”란, Vue.js에서 자주 사용하는 **버튼, 입력창, 테이블, 레이아웃** 등을
### 미리 만들어 제공하는 라이브러리입니다.
### 이를 사용하면 화면을 더 빠르고 쉽게 만들 수 있으며, 디자인도 일관되게 유지할 수 있습니다.

### 예) **Naive UI, Vuetify, Element Plus 등**


## 4. Vue 전용 라이브러리 학습
---
### - Pinia : **Pinia**는 Vue 3의 공식 **상태 관리 라이브러리**로, 컴포넌트 간 데이터를 공유할 때 사용합니다.
### - Vue Router : **Vue Router**는 Vue 애플리케이션에서 **페이지 이동(라우팅)** 을 담당하는 공식 라이브러리입니다.

## 5. 컴포넌트 개발
---
### - 부모 컴포넌트와 자식 컴포넌트의 구조를 이해하고, 이를 활용하여 화면을 구성합니다.
### - Vue Router와 Pinia를 사용하여 페이지 이동과 전역 상태 관리를 구현합니다.
### - 로그인·로그아웃, 세션 유지 등의 기본 기능을 개발 합니다.
### - 여러 개의 페이지를 만들어 SPA(Single Page Application)의 구조를 구현합니다.
